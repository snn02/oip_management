# 2.1.1 Требования к регистрации интересов

## 1. Цели и область действия документа

### 1.1. Цели документа

- Уточнить требования ДРКК "Требования к регистрации интересов" в части правил регистрации и заполнения интересов.
- Сделать процесс регистрации интересов прозрачным и удобным для сотрудников ДРКК Севастополь с учетом локальных требований.

### 1.2. Бизнес-цели

- Повысить качество и точность данных в воронке продаж ЕИС для эффективного управления процессом.
- Создать единую систему классификации услуг, каналов и тегов для упрощения обработки и анализа интересов.

### 1.3. Область действия документа

Документ устанавливает требования к сотрудникам ДРКК Севастополь, применяемые в рамках процесса регистрации клиентских интересов.

## 2. Общие положения

Настоящий документ обязывает ответственных сотрудников ДРКК Севастополь соблюдать установленный порядок регистрации клиентских интересов при осуществлении процесса продажи ПП и услуг.

## 3. Описание процесса

### 3.1. Правила создания нового интереса

Новый интерес должен быть зарегистрирован в следующих случаях:

- При обращении нового клиента.
- При обращении клиента, зарегистрированного в ЕИС, по которому нет активного интереса.
- Если содержание текущего обращения клиента не совпадает с содержанием уже активных интересов.
- Если в потребностях клиента выделяется несколько типов услуг, для каждого из них создаётся отдельный интерес.
- При интересе клиента к новому ПП.
- При передаче клиента в сопровождение.
- При перезаключении договора или подписании дополнительного соглашения о смене условий сотрудничества.

Важно:

- В случае обращения клиента напрямую к сотруднику производственного подразделения данный сотрудник должен в этот же день передать информацию по обращению ответственному МОР для регистрации нового интереса или РОРБ, если сотрудник не может определить, кому из МОР передать информацию.
- Новый интерес также может быть зарегистрирован и другими подразделениями Компании и передан на РОРБ, который, в свою очередь, должен передать интерес ответственному МОР.
- В случае наличия активного интереса, содержание которого относится к текущему обращению клиента, дальнейшая работа ведется в рамках активного интереса.

### 3.2. Заполнение атрибутов интереса

МОР обязан заполнить в карточке интереса поля:

- `Тема` — заполняется по шаблону в соответствии с требованиями к именованию интересов.
- `Клиент`.
- `Контакт` — основное контактное лицо клиента по интересу; если в списке контактных лиц не найдено, следует создать новое.
- `Тип услуги` — определяется классификатором услуг, приведенным в п. 3.2.1.
- `Источник информации` — канал рекламного воздействия, который побудил клиента обратиться в компанию, согласно п. 3.2.3.
- `Теги` — заполняются для типа услуги согласно п. 3.2.2; одновременно может быть указано несколько тегов.

Дополнительные требования:

- В процессе работы с интересом и уточнения потребности до заключения договора следует регулярно пересматривать значение поля `Тип услуги`.
- После заключения договора при изменении типа услуги необходимо руководствоваться правилами регистрации нового интереса.
- Нужно ввести информацию о первом взаимодействии на закладке "Следующие действия" и запланировать следующее взаимодействие.
- Если интерес получен по тендеру, следует включить одноименный флажок и указать срок, до которого принимаются предложения.

### 3.2.3. Классификатор каналов

| Наименование канала | Критерии определения |
| --- | --- |
| Эл.почта | Если обращение поступило по e-mail, указанному на сайте или в других источниках. Менеджер должен уточнить, откуда у клиента данный адрес, при условии начала взаимодействия, и внести корректировки в канал обращения. |
| Элемент из группы "Интернет" | Содержит каналы, которые могут автоматически заполняться во входящем событии ЕИС из форм, связанных с соответствующими источниками. |
| Входящий звонок | Данный канал отсутствует, так как его использование является некорректным. Менеджер, диспетчер должен уточнить у клиента источник, где тот увидел номер телефона, и указать корректный канал обращения. |
| Рекомендация сотрудника | Данный канал предназначен для фиксации сотрудников, передавших нам интерес, с целью определения филиала, проявляющего наибольшую активность в сотрудничестве. Анализируем действия сотрудника, которые способствовали привлечению клиента. |
| Ранее работал с нами | Клиент обратился напрямую в ДРКК Севастополь или в другое подразделение, и нам было передано входящее обращение из другого филиала. При этом у клиента есть успешно закрытые или активные интересы в работе. |
| Отдел диспетчеризации Московский офис | Недопустимо использование. Проводим уточнение у клиента реального канала и источника. В случае отсутствия информации указываем "Без указания источника". |
| Партнерская карточка Битрикс | Необходимо создать партнерскую карточку Битрикс. Использовать для обработки данных, поступающих из Битрикс. |
| Рекомендация партнера | Когда к нам обращаются партнёры, продажи могут осуществляться одним из трех способов: мы реализуем их самостоятельно, по агентской схеме или совместно в рамках проекта. Необходимо обязательно указывать партнера. Не допускается использование для филиалов. |
| От знакомых / коллег | Если клиент обратился напрямую к менеджеру или сотруднику ДРКК Севастополя. Для других филиалов используется канал "Рекомендация сотрудника". |
| Рекомендация клиента | Клиент обратился в офис ДРКК Севастополь. Контактное лицо клиента при взаимодействии назвало другую компанию, с которой любое подразделение ДРКК Севастополь работает или работало ранее. Использование канала без аналитики не допускается. |
| Рекомендация сотрудника клиента | Используется для крупных клиентов, где интересы приходят из разных подразделений. Указываем ответственного сотрудника, кто обратился, для идентификации ответственности за интерес со стороны клиента. Используется только для повторных обращений. |
| Каналы активного привлечения | Группы каналов элементов справочника: PR, обзвон, онлайн мероприятия, очные мероприятия, социальные сети, рассылки. Используем для активных взаимодействий с указанием источника. |

### 3.3. Требования к определению ожидаемой выручки

Для новых интересов ЕИС требуется определять ожидаемую выручку по интересу в соответствии с требованиями к определению ожидаемой выручки по интересам.
