# СМБ-направление ОИП (Service Factory)
# Расшифровка метрик и расчётов

Документ предназначен для руководителя СМБ-направления (Дима)
и описывает:

1. Метрики для отслеживания **KR (Key Results)**
2. Метрики **KPI направления**
3. Дополнительные **управленческие метрики Service Factory**

Цель документа — упростить подготовку регулярных отчётов
и создать единое понимание расчётов.

Горизонт: 2026

---------------------------------------------------------------------

# 1. Метрики для отслеживания Key Results

Эти показатели используются для контроля достижения OKR.

---------------------------------------------------------------------

# Objective 1 — Экономика СМБ-направления

## KR1 — Выполнение финансового плана

Метрика:

    Валовая прибыль СМБ-направления

Формула:

    Валовая прибыль = выручка задач − прямая себестоимость

Себестоимость включает:

- аутсорсинг
- агентские выплаты
- прямые проектные расходы

Цель:

    ≥ 100% плана

Источник:

    ЕИС

---------------------------------------------------------------------

## KR2 — Прибыльность задач

Метрика:

    доля задач с положительной маржой

    прибыльная задача, где количество часов выставленное клиенту \ количество потраченных часов на все взаимодействие по задаче ≥ 80%
    

Формула:

    прибыльные задачи / все закрытые задачи

Пример:

    18 прибыльных задач
    20 закрытых задач

    18 / 20 = 90%

Цель:

    ≥ 90%

---------------------------------------------------------------------

## KR3 — Доля повторных продаж

Метрика:

    доля задач от существующих клиентов

Формула:

    повторные задачи существующих клиентов / все закрытые задачи

Цель:

    ≥ 60%

Почему это важно:

- показатель качества работы
- показатель доверия клиентов
- показатель устойчивости портфеля

---------------------------------------------------------------------

# Objective 2 — Скорость сервиса

## KR1 — Скорость подготовки оценки

Метрика:

    среднее время подготовки оценки между поступлением запроса и отправкой оценки клиенту

Формула:

    сумма времени в рабочих днях подготовки всех оценок / количество оценок

Цель:

    ≤ 1 рабочий день

Источник:

    ЕИС + CRM

---------------------------------------------------------------------

## KR2 — Цикл выполнения задачи

Метрика:

    средний срок выполнения задач

Формула:

    длительность выполнения задачи = дата закрытия − дата начала работы

    средний срок = сумма длительности всех задач / количество задач

Цель:

    ≤ 5 рабочих дней

---------------------------------------------------------------------

## KR3 — Выполнение задач в срок

Метрика:

    доля выполненных в срок задач

    задачей выполненной в срок считается задача, 
    где перенос даты реализации не производился или произведен в связи с расширением требований от заказчика

Формула:

    задачи в срок / все закрытые задачи

Цель:

    ≥ 90%

---------------------------------------------------------------------

# Objective 3 — Поток задач

## KR1 — Поток входящих обращений

Метрика:

    количество входящих обращений за месяц

Источники:

- ЕИС
- CRM

Цель:

    30–50 обращений / месяц

---------------------------------------------------------------------

## KR2 — Подготовленные предложения

Метрика:

    количество отправленных коммерческих предложений

Источник:

    - ЕИС
    - CRM

Цель:

    20–30 предложений / месяц

---------------------------------------------------------------------

## KR3 — Выполненные задачи

Метрика:

    количество закрытых задач за период

Источник:

    - YT

Цель:

    10–20 задач / месяц

---------------------------------------------------------------------

# Objective 4 — Управляемость Service Factory

## KR1 — Прозрачность очереди задач

Факт наличия единой очереди задач.

В системе должны быть статусы:

- зарегистрирована (первичный интерес)
- к обсуждению (выявление потребностей, предложение)
- в очереди (выполнение договора)
- в работе (выполнение договора)
- опубликовано (выполнение договора)
- готово (интерес закрыт, интерес потерян)

Метрика:

    наличие управляемой очереди (да / нет)

Источник:

    YouTrack / ЕИС

---------------------------------------------------------------------

## KR2 — Каталог типовых задач

Факт существования каталога:

- типовые задачи
- типовые оценки
- шаблоны решений

Метрика:

    наличие каталога (да / нет)
    ежемесячно добавляется 5 новых записей в элемент каталога

Источник:

    Репозиторий каталога

---------------------------------------------------------------------

## KR3 — рост входящего потока обращений

Метрика отражает размер входящего потока задач.

Метрика:

    количество входящих СМБ-обращений за месяц

Источник:

    - ЕИС

Цель: 

    30-50 обращений в месяц

---------------------------------------------------------------------

# 2. KPI СМБ-направления

KPI используются для оценки эффективности работы руководителя направления.

Оцениваются **ежеквартально**.

---------------------------------------------------------------------

## KPI 1 — Валовая прибыль направления

Формула:

    Валовая прибыль = выручка − прямая себестоимость

Порог включения премии:

    ≥ 90% плана

---------------------------------------------------------------------

## KPI 2 — Количество выполненных задач

Метрика:

    закрытые задачи / период

Цель:

    ≥ 10 задач / месяц

Этот показатель отражает **throughput Service Factory**.

---------------------------------------------------------------------

## KPI 3 — Доля повторных продаж

Метрика:

    задачи от существующих клиентов / все задачи

Цель:

    ≥ 60%

Показатель отражает:

- качество выполнения задач
- удовлетворённость клиентов
- устойчивость портфеля

---------------------------------------------------------------------

## KPI 4 — Скорость подготовки оценки

Метрика:

    среднее время от обращения клиента до отправки оценки

Цель:

    ≤ 1 рабочий день

Этот показатель отражает **качество клиентского сервиса**.

---------------------------------------------------------------------

# 3. Дополнительные управленческие метрики

Эти показатели **не являются KPI**, но помогают управлять системой.

---------------------------------------------------------------------

## Размер клиентского портфеля

Метрика:

    количество активных клиентов СМБ

Используется для оценки:

- устойчивости бизнеса
- зависимости от отдельных клиентов

---------------------------------------------------------------------

## Конверсия обращений

Метрика:

    выполненные задачи / входящие обращения

Формула:

    конверсия = задачи / обращения

Пример:

    40 обращений
    15 задач

    конверсия = 15 / 40 = 37%

---------------------------------------------------------------------

## Средний чек задачи

Метрика:

    выручка задач / количество задач

Показывает:

- структуру задач
- потенциал роста дохода

---------------------------------------------------------------------

## Средний размер задачи

Метрика:

    часы задач / количество задач

Позволяет понять:

- сложность задач
- границу между СМБ и проектами

---------------------------------------------------------------------

## Средний цикл задачи

Метрика:

    среднее время выполнения задачи

Показывает:

- скорость работы Service Factory
- наличие узких мест

---------------------------------------------------------------------

# 4. Минимальный набор отчёта руководителя СМБ

Для ежемесячного отчёта достаточно 7 показателей:

1. Валовая прибыль
2. Количество выполненных задач
3. Доля повторных продаж
4. Скорость подготовки оценки
5. Поток входящих обращений
6. Throughput задач
7. Средний цикл задачи

Этот набор позволяет быстро оценить состояние Service Factory.
