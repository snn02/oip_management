# Проектное направление ОИП — Расшифровка метрик и расчётов

Документ предназначен для руководителя проектного направления (Виктория) и
описывает:

1. Метрики для отслеживания **KR (Key Results)** по каждому Objective
2. Метрики **KPI направления**
3. **Управленческие метрики** для проектного комитета
4. Минимальный набор **ежемесячного отчёта руководителя направления**

Цель документа — упростить контроль проектов и подготовку регулярных отчётов.

Документ является приложением к `oip_project_strategy_2026.md`.

Горизонт: 2026

---

# 1. Метрики Key Results

Эти показатели используются для контроля достижения OKR проектного направления.
Они не входят в расчёт премии, но показывают, происходят ли нужные изменения в системе.

---

## Objective 1 — Проекты приносят плановую валовую прибыль

---

### KR1 — Выполнение финансового плана

Метрика показывает, генерирует ли проектное направление запланированную прибыль.

Показатель:

    выполнение плана по валовой прибыли проектов

Формула:

    выполнение плана = факт ВП / план ВП

Цель:

    ≥ 100%

Источник данных:

    ЕИС — финансовый план и фактическая валовая прибыль проектов

Почему это важно:

- является главным финансовым результатом проектного направления
- отражает итог работы всего портфеля проектов за квартал

---

### KR2 — Прибыльность проектов

Метрика защищает от ситуации, когда несколько крупных проектов маскируют убытки по другим.

Показатель:

    доля проектов с положительной маржой

Формула:

    прибыльные проекты / все завершённые проекты

Проект считается прибыльным если:

    выручка проекта > прямая себестоимость проекта (в терминах планов проектов ЕИС)

Цель:

    ≥ 90%

Источник данных:

    ЕИС — план и факт по каждому проекту

---

### KR3 — Точность оценки проектов

Метрика показывает зрелость системы управления содержанием и качество оценки трудозатрат.

Показатель:

    отклонение фактических трудозатрат от плановой оценки

Формула:

    отклонение = (факт часы − оценка часов) / оценка часов

Показатель портфеля:

    доля проектов, где |отклонение| ≤ 20%

Цель:

    отклонение ≤ ±20% по портфелю проектов

Источник данных:

    YouTrack — фактические часы  
    ЕИС — плановая оценка трудозатрат проекта

Почему это важно:

- улучшает качество пресейла и снижает риск убыточных проектов
- если показатель ухудшается — анализируется управление требованиями и scope creep

---

### KR4 — Соблюдение сроков проектов

Метрика показывает управляемость сроков проектного портфеля.

Показатель:

    доля проектов, завершённых в согласованный срок

Формула:

    проекты завершённые в срок / все завершённые проекты

Срок проекта соблюдён, если:

    фактическая дата завершения ≤ согласованный срок ± 10%

Цель:

    ≥ 90%

Источник данных:

    паспорт проекта — согласованный срок  
    ЕИС / YouTrack — фактическая дата закрытия проекта

---

## Objective 2 — Управление содержанием, бюджетом, сроками и ожиданиями клиента

---

### KR1 — Формализованное содержание проекта

Метрика проверяет наличие паспорта проекта по каждому активному проекту.

Показатель:

    доля проектов с формализованным содержанием

Формула:

    проекты с паспортом / все активные проекты

Паспорт проекта считается заполненным, если содержит:

- цели проекта
- образ результата
- бюджет в трудозатратах и деньгах
- границы работ
- критерии приёмки результата
- показатели использования результата
- ключевые контрольные точки проекта

Цель:

    100% активных проектов

Источник данных:

    паспорт проекта в системе управления

Почему это важно:

- синхронизирует ожидания заказчика и команды на старте
- является основой для оценки трудозатрат и прогнозирования

---

### KR2 — Прозрачность требований проекта

Метрика проверяет, ведётся ли актуальный реестр требований по каждому проекту.

Показатель:

    доля проектов с актуальной картиной требований

Формула:

    проекты с актуальным реестром / все активные проекты

Реестр считается актуальным, если по каждому проекту известно:

- сколько требований всего
- сколько требований оценено
- сколько требований выполнено
- сколько требований поступило
- сколько требований находится в работе

Цель:

    100% активных проектов

Источник данных:

    YouTrack — реестр требований / задач по проекту

Почему это важно:

- позволяет управлять ростом содержания проекта (scope creep)
- является основой для прогноза бюджета и сроков

---

### KR3 — Управляемость бюджета проекта (EAC)

Метрика проверяет, есть ли по каждому проекту прогноз итоговых трудозатрат.

Показатель:

    доля проектов с актуальным прогнозом трудозатрат до завершения

Формула прогноза (EAC — Estimate at Completion):

    EAC = фактические трудозатраты + оценка оставшихся работ

Цель:

    EAC ведётся по 100% активных проектов и обновляется раз в 2 недели

Источник данных:

    YouTrack

Почему это важно:

- позволяет заранее выявлять риск перерасхода бюджета
- даёт возможность принять управленческое решение до того, как проект уйдёт в минус

---

### KR4 — Точность прогноза сроков проекта

Метрика проверяет точность прогнозирования сроков, а не просто наличие прогноза.
«Поддерживается прогноз» — слабая формулировка, которая позволяет просто переносить дату вперёд.
Нужно измерять качество прогноза.

Показатель:

    точность прогноза даты завершения проекта

Формула:

    отклонение = |прогнозная дата завершения − фактическая дата завершения| / длительность проекта

Цель:

    отклонение ≤ ±20% по портфелю проектов

Источник данных:

    история обновлений прогноза завершения в системе управления проектами

Почему это важно:

- проверяет, понимает ли команда реальный объём оставшихся работ
- если прогноз постоянно переносится — сроками не управляют, а фиксируют факт

---

### KR5 — Управление рисками проекта

Метрика проверяет, ведётся ли реестр рисков по каждому активному проекту.

Показатель:

    доля проектов с актуальным реестром рисков

Формула:

    проекты с реестром рисков / все активные проекты

Риск считается зафиксированным, если:

- он описан
- определён владелец риска
- определён план действий по снижению или принятию риска

Цель:

    100% активных проектов

Источник данных:

    реестр рисков проекта (паспорт проекта или отдельный документ)

Почему это важно:

- критические риски выявляются до того, как приводят к срыву сроков или бюджета
- риски по активным проектам обсуждаются на проектном комитете

---

## Objective 3 — Предсказуемая загрузка производства

---

### KR1 — Уровень загрузки команды

Метрика показывает, насколько эффективно используется производственная команда.

Показатель:

    средняя загрузка специалистов на проектных работах

Формула:

    загрузка = проектные часы / доступные часы команды

Целевой диапазон:

    70–85%

Интерпретация:

    < 70% — недостаточный поток проектов, риск простоев  
    > 85% — перегруз, риск срыва сроков и выгорания команды

Источник данных:

    YouTrack — учёт часов по проектным задачам

---

### KR2 — Прогноз загрузки производства

Метрика проверяет, есть ли актуальный прогноз загрузки по портфелю проектов.

Показатель:

    наличие прогноза загрузки производства до завершения каждого активного проекта

Цель:

    прогноз ведётся и обновляется перед каждым проектным комитетом

Источник данных:

    оценка оставшихся трудозатрат по каждому проекту  
    данные о запускаемых проектах из воронки продаж

Почему это важно:

- позволяет Сербину планировать распределение ресурсов заранее
- исключает неожиданные простои и перегрузки

---

### KR3 — Необъяснённые простои специалистов

Показатель:

    доля простоев специалистов, выделенных на проекты, без зафиксированной причины

Формула:

    часы без задачи (без зафиксированной причины) / доступные часы команды

Цель:

    < 5% рабочего времени

Источник данных:

    YouTrack — задачи и учёт времени

---

## Objective 4 — Масштабируемость и передача в сопровождение

---

### KR1 — Управляемая ёмкость команды

Метрика фиксирует соответствие портфеля реальной пропускной способности команды.

Показатель:

    количество одновременно активных проектов

Целевая ёмкость:

    1 крупный проект + 2 средних проекта одновременно

Добавление проектов сверх ёмкости требует явного согласования с руководителем ОИП.

Источник данных:

    реестр активных проектов

---

### KR2 — Быстрый запуск проектов

Показатель:

    среднее время от подписания договора до старта работ по проекту

Формула:

    срок запуска = дата первой задачи проекта − дата подписания договора

Цель:

    ≤ 5 рабочих дней

Источник данных:

    ЕИС — дата договора  
    YouTrack — дата создания первых задач проекта

Почему это важно:

- быстрый старт снижает риск снижения вовлечённости заказчика и команды
- уменьшает межпроектные простои

---

### KR3 — Стандартизация проектов

Метрика проверяет наличие типовых шаблонов для каждого типа проекта.

Показатель:

    наличие классификатора типов проектов и шаблонов планов

Типы проектов определяются методологией продаж ОИП:

- сайт
- корпоративный портал
- интернет-магазин
- интеграция Битрикс24

Цель:

    для каждого типа проекта существует шаблон плана и типовая оценка трудозатрат  
    для каждой глубины пресейла

Источник данных:

    база знаний / система управления проектами

Почему это важно:

- повышает скорость оценки и снижает трудозатраты на пресейл
- повышает точность первичных оценок

---

### KR4 — Передача проектов в сопровождение

Метрика закрывает жизненный цикл проекта и обеспечивает рост клиентской базы сопровождения.

Показатель:

    доля завершённых проектов с зафиксированным решением о судьбе клиента

Формула:

    проекты с решением / все завершённые проекты

Возможные решения:

- **передан в сопровождение к Ире** — проект закрыт актом, клиент передан
- **завершён без сопровождения** — зафиксирована причина; анализируется в уроках проекта

Цель:

    100% завершённых проектов

Источник данных:

    реестр завершённых проектов  
    протокол передачи в сопровождение

---

### KR5 — Обучение на опыте проектов

Показатель:

    доля завершённых проектов с проведённой ретроспективой

Формула:

    проекты с ретроспективой / все завершённые проекты

Цель:

    100% завершённых проектов

Источник данных:

    база знаний — уроки проектов

# 2. KPI проектного направления

KPI оцениваются **ежеквартально** и влияют на расчёт премии.

> Загрузка производства **не является KPI** — это диагностическая метрика,
> которая смотрится на проектном комитете раз в 2 недели.

| KPI | Показатель | Формула / Метрика | Цель |
|---|---|---|---|
| KPI 1 | Валовая прибыль проектов | Выручка проектов – прямая себестоимость | ≥ 80% плана (порог премии) |
| KPI 2 | Точность оценки проектов | Факт часы / оценка по портфелю | Отклонение ≤ ±20% |
| KPI 3 | Доля проектов, завершённых в срок | Проекты в срок / все завершённые | ≥ 80% |
| KPI 4 | Передача проектов в сопровождение | Проекты с решением о судьбе / завершённые | 100% проектов |

---

### KPI 1 — Валовая прибыль проектов

Ключевой показатель эффективности проектного направления.
Отражает итоговую экономику портфеля проектов за квартал.

Формула:

    ВП = выручка проектов − прямая себестоимость

Себестоимость включает:

- трудозатраты сотрудников
- аутсорсинг и подрядчиков
- прочие прямые расходы проекта

Порог включения премии:

    ВП ≥ 80% от плана

Источник данных:

    ЕИС

---

### KPI 2 — Точность оценки проектов

Показывает качество управления содержанием проектов
и зрелость системы оценки трудозатрат.

Формула:

    отклонение = (факт часы − оценка часов) / оценка часов

Агрегат по портфелю:

    доля проектов с |отклонением| ≤ 20%

Норма:

    ≥ 70% проектов укладываются в ±20%

Если показатель ухудшается — анализируется управление требованиями и scope creep.

Источник данных:

    YouTrack — фактические часы  
    ЕИС — плановая оценка

---

### KPI 3 — Доля проектов, завершённых в срок

Отражает управляемость сроков и способность команды контролировать ход проектов.

Формула:

    проекты завершённые в срок / все завершённые проекты

Норма:

    ≥ 80%

Источник данных:

    паспорт проекта — согласованный срок  
    ЕИС / YouTrack — фактическая дата закрытия

---

### KPI 4 — Передача проектов в сопровождение

Фиксирует завершённость жизненного цикла проекта
и вклад в рост направления сопровождения (Ира).

Метрика:

    по 100% завершённых проектов принято явное решение о дальнейшей работе с клиентом

Варианты решения:

- **передан в сопровождение** — проект закрыт актом, клиент передан команде Иры
- **завершён без сопровождения** — зафиксирована причина; анализируется в уроках проекта

Источник данных:

    реестр завершённых проектов  
    протокол передачи в сопровождение

---

# 3. Управленческие метрики проектного комитета

Проектный комитет проводится **раз в 2 недели**.

Эти показатели **не являются KPI** и не входят в расчёт премии.
Они используются для оперативного управления портфелем и выявления проблем
до того, как они приведут к срыву сроков или бюджета.

---

## 3.1 Сводная таблица проектного комитета

Формат быстрого обзора состояния портфеля на каждом комитете.

| Блок | Показатель | Норма / Интерпретация |
|---|---|---|
| Портфель | Валовая прибыль портфеля | ≥ 80% плана |
| Портфель | Загрузка производства | 70–85% |
| Портфель | Прогноз загрузки (2 мес.) | Наличие прогноза по всем проектам |
| Проект | Coverage (оценённые задачи / все) | ≥ 80% норма; < 70% риск |
| Проект | SPI (Schedule Performance Index) | ≥ 0.9 норма; < 0.9 отставание |
| Проект | CPI (Cost Performance Index) | ≥ 0.9 норма; < 0.9 перерасход |
| Проект | EAC (прогноз итоговых трудозатрат) | EAC ≤ бюджет проекта |
| Проект | Прогноз даты завершения | Отклонение ≤ ±20% от плана |

**Цветовая модель здоровья проекта:**

- 🟢 **Зелёный** — все показатели в норме; проект управляем
- 🟡 **Жёлтый** — один показатель вне нормы; требует внимания
- 🔴 **Красный** — два и более показателя вне нормы; нужны управленческие действия

---

## 3.2 Расшифровка метрик проектного комитета

---

### Coverage — покрытие оценкой

Показывает, насколько содержание проекта декомпозировано и оценено.
Низкий Coverage означает, что оценки ненадёжны и прогноз сроков невозможен.

Формула:

    Coverage = оценённые задачи / все задачи проекта

Интерпретация:

    ≥ 80% — норма, проект хорошо декомпозирован  
    70–80% — допустимо, требует внимания  
    < 70% — риск: прогноз сроков и бюджета ненадёжен

Источник:

    YouTrack — задачи проекта

---

### SPI — Schedule Performance Index

Показывает соответствие фактического выполнения плану по срокам.

Формула:

    SPI = выполненные работы / запланированные работы на дату

Интерпретация:

    > 1.0 — опережение графика  
    = 1.0 — выполняется по плану  
    < 0.9 — отставание, требует внимания  
    < 0.7 — критическое отставание

Источник:

    YouTrack — выполненные задачи и плановый график

---

### CPI — Cost Performance Index

Показывает соотношение плановых и фактических трудозатрат на выполненный объём.

Формула:

    CPI = плановые часы на выполненный объём / фактические часы

Интерпретация:

    > 1.0 — работаем эффективнее плана  
    = 1.0 — по плану  
    < 0.9 — перерасход трудозатрат, требует внимания  
    < 0.7 — критический перерасход

Источник:

    YouTrack — фактические часы  
    ЕИС — плановые трудозатраты

---

### EAC — Estimate at Completion (прогноз итоговых трудозатрат)

Показывает ожидаемую итоговую трудоёмкость проекта.
Используется для прогноза маржи проекта до его завершения.

Формула:

    EAC = фактические трудозатраты + оценка оставшихся работ

Интерпретация:

    EAC ≤ бюджет проекта — проект укладывается в экономику  
    EAC > бюджет проекта — риск перерасхода, нужно управленческое решение

Управленческие решения при EAC > бюджет:

- скорректировать содержание проекта (убрать или отложить требования)
- договориться с заказчиком об изменении бюджета
- принять убыток и зафиксировать урок

Источник:

    YouTrack — фактические часы + оценка оставшихся задач

---

### Forecast Accuracy — точность прогноза даты завершения

Показывает качество управления сроками — насколько рано команда понимает, что срок поедет.

Формула:

    отклонение = |прогнозная дата завершения − фактическая дата завершения| / длительность проекта

Норма:

    |отклонение| ≤ 20% по портфелю

Интерпретация:

    отклонение растёт — команда не видит проблемы заранее  
    отклонение стабильно ≤ 20% — команда управляет сроками

Источник:

    история обновлений прогноза завершения в системе управления

---

## 3.3 Метрики портфеля

---

### Загрузка производства

Показатель:

    проектные часы / доступные часы команды

Формула:

    загрузка = Σ часов по проектам / Σ доступных часов специалистов

Целевой диапазон:

    70–85%

Источник:

    YouTrack — учёт часов

---

### Прогноз загрузки

Показатель:

    плановые часы по активным и запускаемым проектам на ближайшие 2 месяца

Цель:

    прогноз актуален и обновлён перед каждым комитетом

Источник:

    оценка оставшихся работ + воронка продаж

---

## 3.4 Дополнительные диагностические метрики

Смотрятся по необходимости, не на каждом комитете.

---

### Scope Stability — стабильность содержания проекта

Показывает, насколько стабильно содержание проекта после старта разработки.
Нестабильный scope — первопричина перерасходов и срывов сроков.

Формула:

    Scope Change = (текущее количество требований − исходное) / исходное

Интерпретация:

    < 10% — стабильный проект  
    10–25% — умеренные изменения, требует контроля  
    > 25% — риск scope creep, нужно управленческое решение

Если scope нестабилен — анализируется дисциплина фиксации изменений через Change Request.

---

### Возвраты работ на доработку

Метрика:

    задачи возвращённые на доработку / все задачи проекта

Показывает качество постановки задач и качество разработки.

Ориентир:

    ≤ 10%

Источник:

    YouTrack — статусы задач

---

### Средний возраст задач

Выявляет зависшие задачи, которые тормозят движение проекта.

Формула:

    среднее (текущая дата − дата начала задачи) по открытым задачам

Источник:

    YouTrack

---

### НЗП — незавершённое производство проекта

Показывает долю работ, выполненных, но ещё не принятых заказчиком.
Высокий НЗП означает риск задержки актирования и финансовых потерь.

Формула:

    НЗП = часы в незакрытых задачах / общие часы проекта

Ориентир:

    ≤ 15%

Источник:

    YouTrack — часы в задачах по статусу

---

### Размер активного портфеля

Метрика:

    количество активных проектов

Используется для контроля нагрузки РП и соответствия ёмкости команды.

Ориентир:

    1 крупный + 2 средних проекта одновременно
