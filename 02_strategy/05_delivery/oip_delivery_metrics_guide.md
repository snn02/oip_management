# Группа разработки ОИП — Расшифровка метрик и расчётов

Документ предназначен для руководителя группы разработки (Александр Сербин) и описывает:

1. Метрики для отслеживания **KR (Key Results)** по каждому Objective
2. Метрики **KPI группы**
3. **Управленческие метрики** для производственного комитета
4. Минимальный набор **ежемесячного отчёта руководителя группы**

Цель документа — упростить управление производством и подготовку регулярных отчётов.

Горизонт: 2026

# 1. Метрики Key Results

Эти показатели используются для контроля достижения OKR группы разработки.
Они не входят в расчёт премии, но показывают, происходят ли нужные изменения в системе.

---

## Objective 1 — Группа разработки работает как единое управляемое производство

---

### KR1 — Управляемость загрузки группы

Метрика показывает, насколько эффективно используется производственный ресурс группы
по всем направлениям в совокупности.

Показатель:

    средняя загрузка специалистов группы

Формула:

    загрузка = продуктивные часы / доступные часы группы

Продуктивные часы включают:

- коммерческие часы: задачи направлений (проекты, СМБ, сопровождение)
- внутренние проекты и инициативы
- обучение и развитие по ИПР (по факту, но не более норматива)

Целевой диапазон:

    70–85%

Интерпретация:

    < 70% — недостаточный поток задач или проблема планирования
    > 85% — риск перегрузки, срыва сроков, выгорания команды

Источник данных:

    ЕИС — учёт часов по специалистам и направлениям

Почему это важно:

- по каждому специалисту в любой момент должны быть известны:
  текущее базовое направление, активные задачи,
  уточнённая загрузка на 2 недели и плановая на 2 месяца,
  доступные потребности для привлечения к другим направлениям
- исключает ситуацию, когда баланс загрузки между направлениями
  существует только «в голове» руководителя

---

### KR2 — Снижение необъяснённых простоев

Показатель:

    доля вынужденных простоев специалистов без зафиксированной причины

Формула:

    часы «вынужденного простоя» без зафиксированной причины / доступные часы группы

Цель:

    < 5% рабочего времени

Простой считается объяснённым, если зафиксирована одна из причин:

- ожидание задачи от направления (с датой ожидаемого поступления)
- технические проблемы: нет света, интернета, доступа к ресурсам разработки
- плановое отсутствие

Источник данных:

    ИСУП — учёт часов и причин простоев

---

### KR3 — Управляемый вход задач

Метрика проверяет, соблюдается ли правило принятия задач в разработку.

Показатель:

    доля задач, принятых в работу с соблюдением правил входа

Формула:

    задачи с полным набором параметров / все задачи, принятые в работу

Задача считается принятой корректно, если зафиксированы:

- цель задачи
- критерии результата
- оценка трудоёмкости
- согласованный срок

Цель:

    ≥ 80% задач

Источник данных:

    YouTrack — описание задач на момент перехода в статус «В работе»

Почему это важно:

- задачи без критериев приводят к возвратам и перерасходу
- группа разработки является соавтором постановки, а не только получателем

---

## Objective 2 — Группа разработки обеспечивает предсказуемое выполнение задач

---

### KR1 — Точность оценок

Метрика показывает зрелость системы оценки трудозатрат и качество декомпозиции.

Показатель:

    отклонение фактических трудозатрат от согласованной оценки

Формула:

    отклонение = (факт часы − оценка часов) / оценка часов

Показатель портфеля:

    доля задач, где |отклонение| ≤ 20%

Цель:

    ≥ 80% задач укладываются в ±20%

Источник данных:

    YouTrack — фактические часы и плановая оценка задачи

Почему это важно:

- перерасход трудозатрат напрямую снижает валовую прибыль направлений
- систематическое превышение оценок — сигнал проблемы с декомпозицией или пониманием задачи

---

### KR2 — Качество выполнения задач

Метрика показывает техническое качество разработки.

Показатель:

    доля задач без возврата на доработку по вине разработки

Формула:

    задачи без возврата / все выполненные задачи

Возврат учитывается, если причина — ошибка разработки или неверное понимание
критериев результата со стороны группы разработки.

Возвраты по причине изменения требований заказчиком — не учитываются.

Цель:

    ≥ 80%

Источник данных:

    YouTrack — задачи с причиной возврата «ошибка разработки»

---

### KR3 — Качество задач на входе

Метрика показывает, насколько задачи поступают в разработку с корректной постановкой.

Показатель:

    доля задач, соответствующих критериям хорошей задачи

Задача соответствует критериям, если содержит:

- понятную цель
- зафиксированный критерий результата
- обновляемый план реализации
- реалистичную оценку трудоёмкости
- согласованный срок

Формула:

    задачи с критериями / все задачи периода

Цель:

    ≥ 80%

Источник данных:

    YouTrack — поля задач

Примечание:

    Этот показатель — совместная ответственность группы разработки
    (активное участие в постановке, инициирование уточнений)
    и направлений-заказчиков (качество формирования задач).

---

### KR4 — Экономическая эффективность разработки

Метрика показывает реальную отдачу от каждого доступного часа группы.

Показатель:

    нормированная валовая прибыль группы

Формула:

    нВП = валовая прибыль группы / доступные часы группы

Цель:

    нВП ≥ плановому уровню

Источник данных:

    ЕИС — валовая прибыль по группе, доступные часы группы

Почему это важно:

- при высокой загрузке и низкой нВП — проблема точности оценок или перерасходов
- при низкой загрузке и нормальной нВП — проблема потока задач, а не эффективности

---

## Objective 3 — Специалисты решают задачи самостоятельно в рамках закреплённых направлений

---

### KR1 — Самостоятельность в закреплённых направлениях

Метрика проверяет, формализован ли уровень самостоятельности специалистов.

Показатель:

    наличие зафиксированного уровня самостоятельности для каждого специалиста

Считается выполненным, если для каждого специалиста определены:

- перечень типов задач, которые он выполняет без эскалации к руководителю разработки
- перечень ситуаций, требующих согласования
- план развития самостоятельности

Цель:

    100% специалистов имеют зафиксированный уровень самостоятельности

Источник данных:

    ИПР специалиста / матрица квалификации / треки развития

---

### KR2 — Комплексность решения задач

Метрика показывает, снижается ли зависимость между специалистами при выполнении задач.

Показатель:

    доля задач, переданных между специалистами из-за узкой специализации

Формула:

    задачи с «вынужденной» передачей / все задачи периода

Вынужденная передача — передача задачи другому специалисту,
потому что первый не владеет нужной компетенцией,
а не по причине плановой распределённой работы.

Цель:

    тренд снижения квартал к кварталу

Источник данных:

    YouTrack — история смены исполнителя с причиной

---

### KR3 — Индивидуальные планы развития

Метрика проверяет наличие и актуальность ИПР.

Показатель:

    доля специалистов с актуальным ИПР

Формула:

    специалисты с актуальным ИПР / все специалисты группы

ИПР считается актуальным, если:

- цели развития соответствуют закреплённому направлению
- соответсвует критериям хорошего ИПР
- обновлён не более квартала назад
- включает конкретные шаги через практику и совместную работу:
  парное решение задач, внутренние ревью, разбор сложных случаев

Цель:

    100% специалистов

Источник данных:

    база ИПР

---

## Objective 4 — Развитие инфраструктуры и производительности разработки

---

### KR1 — Развитие производственной инфраструктуры

Метрика отслеживается раздельно по трём контурам.

**Контур ИБ и соответствия регуляторам:**

Показатели:

    отсутствие критических инцидентов по безопасности
    выполнение требований к обработке ПДн

Цель:

    критические риски ИБ устранены; соответствие требованиям к ПДн обеспечено

**Контур Платформы разработки:**

Показатели:

    время развёртывания серверного и локального окружения разработки
    наличие документированного процесса деплоя

Цель:

    время развёртывания окружения снижено до 30 мин
    процесс деплоя стандартизирован и задокументирован

**Контур Операционной стабильности:**

Показатели:

    количество операционных инцидентов за период
    среднее время устранения инцидента

Цель:

    тренд снижения инцидентов
    время устранения типового инцидента ≤ согласованного SLA

Источник данных:

    реестр инцидентов (YouTrack)
    подробный план инфраструктурных работ: https://drive.rarus.ru/apps/onlyoffice/76956263

---

### KR2 — Рост производительности разработки

Метрика фиксирует измеримое улучшение хотя бы по одному показателю
производительности за год.

Показатели (выбирается один или несколько для отслеживания в начале года):

    среднее время выполнения задачи от постановки до закрытия
    средние трудозатраты на типовую задачу по направлению
    доля задач с возвратом на доработку

Цель:

    положительная динамика по сравнению с базовым значением Q1 2026

Источник данных:

    YouTrack — данные по задачам за квартал

Примечание:

    Технологические инициативы (AI-инструменты, автоматизация, стандарты разработки)
    являются инструментом достижения этого KR.
    Детальный план инициатив: https://drive.rarus.ru/apps/onlyoffice/76956263

# 2. KPI группы разработки

KPI оцениваются **ежеквартально** и влияют на расчёт премии.

> Загрузка в разрезе направлений, прогноз ВП, доля самостоятельных задач
> и статус инфраструктуры — диагностические метрики производственного комитета,
> не входящие в расчёт премии.

| KPI | Показатель | Формула / Метрика | Цель |
|---|---|---|---|
| KPI 1 | Валовая прибыль группы разработки | Выручка – прямая себестоимость трудозатрат | ≥ 80% плана (порог премии) |
| KPI 2 | Нормированная валовая прибыль | ВП / доступные часы группы | Плановый уровень |
| KPI 3 | Точность оценок | Факт часы / оценка, доля задач ≤ ±20% | ≥ 70% задач |
| KPI 4 | Качество выполнения задач | Задачи без возврата / все задачи | ≥ 80% |
| KPI 5 | Доля задач с критериями хорошей задачи | Задачи с критериями / все задачи | ≥ 80% |

---

### KPI 1 — Валовая прибыль группы разработки

Ключевой показатель эффективности производства.
Отражает итоговую экономику группы по всем направлениям за квартал.

Формула:

    ВП = выручка − прямая себестоимость трудозатрат

Себестоимость включает:

- трудозатраты штатных специалистов группы
- привлечённых подрядчиков и аутсорс

Порог включения премии:

    ВП ≥ 80% от плана

Источник данных:

    ЕИС — финансовый план и факт по группе

---

### KPI 2 — Нормированная валовая прибыль

Показывает реальную эффективность использования производственного ресурса.
Позволяет разделить падение ВП из-за простоев и из-за перерасхода.

Формула:

    нВП = валовая прибыль / доступные часы группы

Интерпретация:

    нВП в норме + загрузка низкая → проблема потока задач (нет работы)
    нВП ниже нормы + загрузка в норме → проблема эффективности (перерасход оценок)

Источник данных:

    ЕИС — валовая прибыль
    ИСУП — доступные часы группы

---

### KPI 3 — Точность оценок

Показывает зрелость системы оценки трудозатрат.
Является ранним индикатором проблем с декомпозицией и пониманием задач.

Формула:

    отклонение = (факт часы − оценка часов) / оценка часов

Агрегат по портфелю:

    доля задач с |отклонением| ≤ 20%

Норма:

    ≥ 70% задач укладываются в ±20%

Если показатель ухудшается — анализируется:

- качество постановки задач и наличие критериев результата
- участие группы разработки в формировании оценок
- декомпозиция сложных задач

Источник данных:

    YouTrack — фактические часы и плановая оценка

---

### KPI 4 — Качество выполнения задач

Отражает техническое качество разработки и соответствие результата критериям приёмки.

Формула:

    задачи без возврата по вине разработки / все выполненные задачи

Норма:

    ≥ 80%

Источник данных:

    YouTrack — задачи с причиной возврата

---

### KPI 5 — Доля задач с критериями хорошей задачи

Отражает управляемость входа задач в разработку и дисциплину постановки.

Задача соответствует критериям, если содержит:

- понятную цель
- зафиксированный критерий результата
- обновляемый план реализации
- реалистичную оценку трудоёмкости
- согласованный срок

Формула:

    задачи с полным набором критериев / все задачи

Норма:

    ≥ 80%

Источник данных:

    YouTrack — поля задач

# 3. Управленческие метрики производственного комитета

Производственный комитет проводится **раз в 2 недели**.

Эти показатели **не являются KPI** и не входят в расчёт премии.
Они используются для оперативного управления группой и раннего выявления проблем.

---

## 3.1 Сводная таблица производственного комитета

| Блок | Показатель | Норма / Интерпретация |
|---|---|---|
| Экономика | Валовая прибыль группы | ≥ 80% плана |
| Экономика | Нормированная ВП | Плановый уровень |
| Загрузка | Средняя загрузка группы | 70–85% |
| Загрузка | Загрузка в разрезе направлений | Нет перегруза в одном при простое в другом |
| Загрузка | Прогноз загрузки на 2 недели / 2 месяца | Актуален по каждому специалисту |
| Загрузка | Доля вынужденных простоев без причины | < 5% |
| Качество | Точность оценок | ≥ 70% задач в ±20% |
| Качество | Возвраты на доработку | < 20% задач |
| Качество | Доля задач с критериями хорошей задачи | ≥ 80% |
| Самостоятельность | Доля задач без эскалации к руководителю разработки | Тренд роста |
| Инфраструктура | Инциденты ИБ | 0 критических |
| Инфраструктура | Операционные инциденты | Тренд снижения |
| Инициативы | Статус технологических инициатив и инфраструктурных контуров | По плану / отставание |

**Цветовая модель здоровья группы:**

- 🟢 **Зелёный** — все показатели в норме; группа управляема
- 🟡 **Жёлтый** — один показатель вне нормы; требует внимания
- 🔴 **Красный** — два и более показателя вне нормы; нужны управленческие действия

---

## 3.2 Расшифровка ключевых метрик комитета

---

### Загрузка группы

Показывает, насколько производственный ресурс используется эффективно.

Формула:

    загрузка = продуктивные часы / доступные часы группы

Продуктивные часы: коммерческие задачи + внутренние инициативы + обучение по ИПР.

Целевой диапазон:

    70–85%

Интерпретация:

    < 70% — недостаточный поток задач, координация с направлениями
    > 85% — риск перегрузки и срыва сроков

Источник:

    ИСУП

---

### Загрузка в разрезе направлений

Показывает баланс загрузки между направлениями.
Помогает принимать решения о перераспределении специалистов.

Формула:

    загрузка по направлению = часы по задачам направления / доступные часы закреплённых специалистов

Интерпретация:

    дисбаланс: одно направление перегружено, другое простаивает →
    рассмотреть перераспределение специалиста из базового направления

Источник:

    ИСУП — часы в разбивке по направлениям

---

### Точность оценок (по задачам)

Ранний индикатор проблем с декомпозицией и постановкой задач.

Формула:

    отклонение = (факт часы − оценка часов) / оценка часов

Интерпретация:

    систематический перерасход (+20% и выше) → проблема постановки или декомпозиции
    систематическое недоиспользование (−20% и ниже) → завышение оценок или незакрытые работы

Источник:

    YouTrack

---

### Возвраты на доработку

Показывает качество разработки и дисциплину проверки результата.

Формула:

    задачи, возвращённые на доработку по вине разработки / все выполненные задачи

Интерпретация:

    > 20% — системная проблема: анализируем критерии приёмки и процесс ревью

Источник:

    YouTrack — задачи со статусом возврата и причиной

---

### Доля задач без эскалации к руководителю разработки

Показывает рост самостоятельности специалистов.
Растущий тренд означает, что руководитель освобождается от операционного посредничества.

Формула:

    задачи, выполненные без обращений к руководителю разработки / все задачи периода

Интерпретация:

    тренд роста — специалисты становятся более самостоятельными
    тренд снижения — либо растёт сложность задач, либо не хватает компетенций

Источник:

    фиксация эскалаций в YouTrack или отдельном журнале

---

### Инфраструктурные инциденты

Отслеживается раздельно по контурам.

**ИБ и соответствие регуляторам:**

    цель: 0 критических инцидентов по безопасности

**Операционная стабильность:**

Показатели:

    количество операционных инцидентов за период
    среднее время устранения

Интерпретация:

    тренд снижения — инфраструктура становится стабильнее
    высокое среднее время устранения — отсутствует документация или стандартные процедуры

Источник:

    реестр инцидентов

---

## 3.3 Дополнительные диагностические метрики

Смотрятся по необходимости, не на каждом комитете.

---

### Прогноз ВП на конец квартала

Показывает, выполняется ли финансовый план при текущей динамике.

Формула:

    прогноз ВП = факт ВП + (нВП × запланированные часы)

Интерпретация:

    прогноз < 80% плана → нужно управленческое решение до конца квартала:
    увеличить поток задач или снизить перерасход оценок или сокращать персонал

Источник:

    ЕИС

---

### Средний возраст задач в работе

Выявляет зависшие задачи, которые тормозят поток и создают НЗП.

Формула:

    среднее (текущая дата − дата постановки задачи) по открытым задачам в статусе «В работе»

Ориентир:

    задачи старше 10 рабочих дней требуют внимания

Источник:

    YouTrack

---

### НЗП — незавершённое производство

Показывает долю трудозатрат, уже вложенных, но ещё не принятых и не актированных.
Высокий НЗП — риск задержки выручки и финансовых потерь при изменении приоритетов.

Формула:

    НЗП = часы в задачах статуса «выполнено / на проверке» без акта / общие часы периода

Ориентир:

    ≤ 15%

Источник:

    YouTrack — часы в задачах по статусам

---

### Cycle Time — время выполнения задачи

Показывает скорость потока разработки: от постановки задачи до её закрытия.

Формула:

    Cycle Time = дата закрытия задачи − дата постановки задачи

Агрегат:

    среднее и медиана по задачам периода

Интерпретация:

    растущий Cycle Time → задачи зависают; ищем причину в блокерах или декомпозиции
    короткий Cycle Time + высокий Throughput → поток здоровый

Источник:

    YouTrack — даты изменения статусов

---

### Throughput — пропускная способность

Показывает, сколько задач группа доводит до результата за период.

Формула:

    Throughput = количество закрытых задач / период (неделя или месяц)

Интерпретация:

    стабильный или растущий Throughput при снижении НЗП — поток здоровый
    падающий Throughput — задачи зависают или поток не поступает

Источник:

    YouTrack — задачи со статусом «Закрыта» за период
